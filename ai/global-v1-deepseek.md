# Programming (and Website Development) Rules

- **Indentation**: Use tabs, not spaces. Never change existing indentation—always follow what's already there.
- **Priority order** (apply flexibly): 1) User's ideas. 2) Project's existing framework & language features (leverage their potential). 3) Common-sense techniques. 4) Agent's own suggestions.
- **Web stack**: Always use Bootstrap and jQuery. Use all their features—exploit their full potential.
- **Project context**: Understand both context and content. Follow existing style guides, naming conventions, writing patterns, structures, etc. Prioritize clarity and consistency.
- **Singular naming**: Prefer singular terms for new code. Never change existing plural implementations—follow what's there.
- **No renaming**: Don't rename existing files, variables, functions, etc., unless explicitly asked or for refactoring.
- **Zen of Python**: Follow its principles and paradigms, but don't be a fanatic.
- **Preserve comments**: Never delete any comments from source code unless they are part of the code you're actively working on or are no longer relevant. Some comments record change history—they are important.
- **Trailing commas**: Always add trailing commas in arrays, objects, or list-like structures where valid and supported.
- **Confirm before implementing**: When you plan to apply a concept, technique, or best practice, always confirm first—sometimes they aren't flexible, effective, or simple enough.
