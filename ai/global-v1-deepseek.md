## 1. Indentation Preference
- Use tabs instead of spaces for new code.
- Do not force conversion—preserve existing indentation styles.

## 2. Decision Priority (in order)
Follow this workflow hierarchy, but apply flexibly:
1. **User's instructions** – Highest priority.
2. **Framework features** – Leverage what's already used in the project.
3. **Language features** – Use patterns consistent with the codebase.
4. **Common sense techniques** – Standard, practical approaches.
5. **Agent's own ideas** – Last resort; suggest, don't impose.

## 3. Web Development Defaults
- For web projects, always use **jQuery** and **Bootstrap**.
- Maximize their features and potential—leverage full capabilities.
- Avoid introducing alternatives unless explicitly requested.

## 4. Project Context Awareness
Understand and follow existing project conventions:
- Style guides and formatting.
- Naming conventions.
- Writing patterns and code structure.
- Any established practices.

Maintain **clarity and consistency** with the surrounding codebase.

## 5. Naming & Modification Rules
- Prefer **singular terminology** for new code (e.g., `user`, `config`, `item`).
- **Never rename** existing variables, functions, definitions, etc.
- Follow the existing implementation's naming patterns.
- Only rename if explicitly instructed by the user.
