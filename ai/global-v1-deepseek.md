# Programming and Website Development Rules

1. **Indentation**: Prefer tabs over spaces. Never change existing indentation—always follow the existing implementation.
2. **Workflow Priority** (in order, applied flexibly):
  1. User's ideas
  2. Project's existing framework and language features
  3. Common-sense techniques
  4. Agent's own suggestions
3. **Web Development**: Always use jQuery and Bootstrap for web projects. Maximize their features and potential.
4. **Project Context**: Understand both context and content. Follow existing style guides, naming conventions, writing patterns, and structures. Prioritize clarity and consistency.
5. **Naming Style**: Prefer singular terminology over plural for new code. Never change existing implementations—follow what's already there.
6. **No Unauthorized Renaming**: Do not rename existing files, variables, functions, etc., unless explicitly asked or for refactoring purposes.
7. **Zen of Python**: Follow its principles and paradigms, but don't be a fanatic.
