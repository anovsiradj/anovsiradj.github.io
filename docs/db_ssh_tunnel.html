Laravel SSH tunnel MySQL connections drop because the tunnel times out due to inactivity, causing the "MySQL server has gone away" error (2006). To fix this, ensure the tunnel is persistent using autossh to auto-restart it, and increase MySQL's wait_timeout or use PDO::ATTR_PERSISTENT => false .

Common Solutions for Persistent Connections  
  • Use Autossh for Tunneling: Instead of standard ssh -L , use autossh to keep the tunnel alive automatically if it dies.
    bash
    autossh -M 0 -f -N -L 33060:127.0.0.1:3306 user@remote-server -o "ServerAliveInterval 30" -o "ServerAliveCountMax 3"
    Use code with caution.
  • Update Laravel DB Configuration: Add PDO::ATTR_PERSISTENT => false to your config/database.php options to force a fresh connection if the old one breaks.
  • Adjust MySQL wait_timeout : Increase the wait_timeout and interactive_timeout on your remote MySQL server (e.g., to 8 hours) to prevent it from closing idle connections.
  • Schedule Tunnel Health Check: Use Laravel Scheduler to run a command that checks if the port is open and restarts the tunnel if needed.

Why This Happens  
  • Idle Tunnels: SSH tunnels die when inactive for a while, and Laravel does not automatically reconnect an established PDO connection.
  • MySQL 8/Cloud Issues: Remote DB servers (like AWS Aurora) often have low wait_timeout settings, dropping connections quickly.
  • Queue Workers: Long-running queue:work processes often experience this because they keep a connection open for a long time.
